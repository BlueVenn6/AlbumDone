# Mobile First Device Test Checklist

## 首次启动

- [ ] 步骤：首次安装应用后直接打开首页
- [ ] 预期结果：应用正常进入首页，底部导航完整，无红屏/白屏
- [ ] 失败记录：平台、机型、系统版本、首屏报错截图

## 权限申请

- [ ] 步骤：首次进入需要照片访问的页面并触发权限请求
- [ ] 预期结果：出现系统照片权限弹窗，文案与系统行为正常
- [ ] 失败记录：是否弹窗、弹窗文案、控制台/原生日志

## 相册列表

- [ ] 步骤：授权后进入相册列表页
- [ ] 预期结果：相册列表可加载，加载中与失败状态可区分
- [ ] 失败记录：页面截图、错误提示文案、设备相册数量

## 进入相册

- [ ] 步骤：点击任意一个非空相册进入详情
- [ ] 预期结果：照片网格正常展示，可继续进入后续处理链路
- [ ] 失败记录：目标相册名称、首个失败图片 URI、页面报错

## 去重分析

- [ ] 步骤：在相册内触发去重分析
- [ ] 预期结果：出现明确加载状态，分析完成后结果可见；失败时有明确错误
- [ ] 失败记录：相册规模、失败阶段、错误提示、日志

## 筛选页

- [ ] 步骤：从首页进入筛选/精选页
- [ ] 预期结果：页面能进入，空态、加载态、失败态清晰
- [ ] 失败记录：入口路径、页面截图、控制台错误

## 截图 OCR

- [ ] 步骤：选择一张截图并触发 OCR
- [ ] 预期结果：图片读取成功，OCR 成功或出现明确“图片读取失败 / base64 转换失败 / OCR 失败”提示
- [ ] 失败记录：平台、截图 URI（`content://` 或 `ph://`）、错误提示、日志

## 截图 AI 指令

- [ ] 步骤：在截图识别成功后执行一次 AI 指令
- [ ] 预期结果：AI 处理成功或出现明确“AI 处理失败”提示，不出现静默卡住
- [ ] 失败记录：provider、model、是否已配置 key、错误提示、日志

## 设置页保存

- [ ] 步骤：在设置页保存 provider、model、baseUrl 等非敏感配置，并输入会话内 API key
- [ ] 预期结果：非敏感配置保存成功；UI 显示“已配置”；真实 key 不回填到持久化 store
- [ ] 失败记录：保存前后页面状态、提示文案、重启前截图

## 重启后配置保留

- [ ] 步骤：完全关闭 App 后重新打开设置页
- [ ] 预期结果：provider、model、baseUrl、supportsVision、hasApiKey 等非敏感状态符合设计；真实 key 不自动回填
- [ ] 失败记录：重启前后配置对比、是否误回填 key、日志

## 无 Key 时提示

- [ ] 步骤：清空当前会话内 key 后直接执行 AI 相关操作
- [ ] 预期结果：出现明确缺少 key 或无法调用 AI 的提示
- [ ] 失败记录：触发入口、提示文案、是否出现误导性的“已配置”

## Android `content://` 验证

- [ ] 步骤：在 Android 真机选择相册图片与截图，覆盖 `content://` URI 场景
- [ ] 预期结果：读取链路能继续执行；失败时出现明确阶段性错误，不静默失败
- [ ] 失败记录：ROM、Android 版本、URI 示例、失败阶段

## iOS `ph://` 验证

- [ ] 步骤：在 iPhone 真机选择照片与截图，覆盖 `ph://` URI 场景
- [ ] 预期结果：读取链路能继续执行；失败时出现明确阶段性错误，不静默失败
- [ ] 失败记录：iOS 版本、URI 示例、失败阶段、原生日志

## 连接测试

- [ ] 步骤：在已配置 provider/baseUrl 的前提下执行一次完整 OCR + AI 链路
- [ ] 预期结果：设备可访问目标服务；若不可访问，错误能反映连接或请求失败
- [ ] 失败记录：网络环境、baseUrl、provider、错误文案、抓包/日志

## 权限拒绝后恢复

- [ ] 步骤：先拒绝照片权限，再从系统设置手动重新授予后回到 App
- [ ] 预期结果：页面能恢复加载相册，不需要卸载重装
- [ ] 失败记录：恢复路径、是否需要重启 App、页面状态截图
